###############################################################################
# Generating Mesh/Coordinates for the Cross Sectional Domain of the Pacific
###############################################################################

Set desired resolution:
  order k=2, level l=1: node dist. ~60km, depth from ~3,000 km to ~1,000 km
  order k=2, level l=2: node dist. ~30km, depth from ~1,000 km to   ~700 km
  order k=2, level l=3: node dist. ~15km, depth from   ~700 km to   ~200 km
  order k=2, level l=4: node dist.  ~7km, depth from   ~200 km to      0 km

Find input options:
[Domain]
  box-subdivision-x = 82  # 230.6 deg (target for domain size is 231.4 deg)
  box-subdivision-y = 1
  box-subdivision-z = 16
[Discretization]
  order = 2
  level-min = 0
  level-max = 6
[Discretization:AMR]
  init-refine = depth
  init-refine-depth = 1000.0e3, 700.0e3, 205.0e3
  init-refine-level-min = 1
  init-refine-level-max = 4

Output mesh has sizes:
  Number of elements: 504136

###############################################################################

Apply rotation to the coordinates such that the cross section is aligned with
the equator with longitude [-theta,+theta]. This is performed by setting two
rotation angles in function `rhea_discretization_write_cont_coordinates`:
  const double        rot_angle[3] = {0.5*M_PI, NAN, 0.5*M_PI};

###############################################################################

Note: The list of refinement depths can be found with the included script,
where the input level is l+1+4 due to the second order discretization k=2 and
box-subdivision-z=16:
  $ ./list_depths.pl 9 210
  $ ./list_depths.pl 8 700
  $ ./list_depths.pl 7 1000
  $ ./list_depths.pl 6 2867
